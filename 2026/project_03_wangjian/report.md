# Final Report Draft: SAM 2 Automatic Mask Generation for Multi-Class Dataset Annotation

## 1. Introduction

This project studies how to use SAM 2 for automatic mask generation and dataset annotation. Since SAM 2 produces high-quality masks but does not assign semantic class labels by itself, we combine YOLOv11 and SAM 2 into a practical annotation pipeline. YOLOv11 predicts object categories and bounding boxes, and SAM 2 uses these bounding boxes as prompts to produce pixel-level instance masks.

The target dataset contains five common desktop object classes: cup, bottle, bowl, book, and cell phone.

## 2. Dataset

The completed dataset is Desktop100, a 100-image segmentation dataset generated from Ultralytics COCO128-seg. We first filter images that contain at least one of the five target classes, then apply deterministic augmentations including horizontal flip and brightness changes. Segmentation polygons are transformed consistently, producing COCO-style ground truth annotations.

The generated dataset is stored locally under `data/desktop100` and ignored by Git. It can be reproduced by running `src/prepare_desktop100_from_coco128.py`.

## 3. Method

The method contains four stages:

1. YOLOv11 detects the five target categories and outputs class labels, confidence scores, and bounding boxes.
2. SAM 2 receives each YOLO bounding box as a box prompt and predicts the corresponding instance mask.
3. The system exports a COCO-style JSON file containing image metadata, category IDs, bounding boxes, and mask polygons.
4. Visualizations are generated by overlaying masks, bounding boxes, and class names on the original image.

This design separates semantic recognition and mask generation. YOLOv11 handles class prediction, while SAM 2 focuses on accurate object boundaries.

## 4. Experiments

The main inference script is `src/main.py`. It can run on the provided examples:

```bash
python src/main.py --input data/test_examples --output results
```

For the full dataset with SAM 2:

```bash
python src/main.py --input data/desktop100/images --output results --conf 0.20 --imgsz 640 --device cuda
```

For evaluation:

```bash
python src/evaluate_coco.py --pred results/annotations/auto_annotations_coco.json --gt data/desktop100/ground_truth_coco.json --output results/tables
```

## 5. Evaluation Metrics

The manually labeled subset is used to compute:

- Intersection over Union (IoU)
- Dice/F1 score
- Precision
- Recall

The final report should include both overall metrics and per-class metrics.

## 6. Results and Analysis

The full YOLOv11 + SAM 2 pipeline has been verified on 100 Desktop100 images. The generated outputs include visualized annotations, a COCO-style annotation file, a detection summary table, and per-class metrics.

| Class | IoU | Dice/F1 | Precision | Recall |
| --- | ---: | ---: | ---: | ---: |
| book | 0.321 | 0.383 | 0.722 | 0.374 |
| bottle | 0.348 | 0.445 | 0.511 | 0.655 |
| bowl | 0.382 | 0.450 | 0.644 | 0.517 |
| cell phone | 0.000 | 0.000 | 0.000 | 0.000 |
| cup | 0.455 | 0.520 | 0.731 | 0.524 |
| overall | 0.340 | 0.403 | 0.644 | 0.447 |

The strongest categories are cup and bowl, where object shapes are clear and YOLOv11n often provides useful prompts. The cell phone class fails in this experiment because the detector misses the small phone instances. This confirms a key limitation of the pipeline: SAM 2 can create high-quality masks when prompted correctly, but it cannot recover objects that the detector does not prompt.

## 7. Conclusion

YOLOv11 + SAM 2 is a practical and extensible pipeline for automatic dataset annotation. It can significantly reduce the effort of manual mask labeling. However, the final annotation quality still depends on the object detector. Future work could use a larger detector, fine-tune YOLO on the collected images, or add an active-learning loop for human review.
