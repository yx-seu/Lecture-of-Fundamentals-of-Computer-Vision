# LeNet-5 MNIST 手写数字识别 FPGA 加速器 -- 演示报告

## 演示概述

在 Alinx AX7010 开发板（Xilinx Zynq-7010, xc7z010clg400-1）上实现了基于 LeNet-5 卷积神经网络的 MNIST 手写数字识别硬件加速器。PS 端（ARM Cortex-A9）负责控制逻辑和 UART 通信，PL 端（FPGA）完成神经网络推理加速，采用纯 AXI4-Lite 总线通信。

## 硬件环境

| 项目 | 规格 |
|------|------|
| 开发板 | Alinx AX7010 |
| FPGA 芯片 | XC7Z010-1CLG400C |
| PS 处理器 | 双核 ARM Cortex-A9 @ 667MHz |
| PL 逻辑资源 | 28K Logic Cells, 17.6K LUTs, 80 DSP, 60 BRAM |
| PL 时钟频率 | 100 MHz |
| 推理延迟 | 约 4.2ms / 张 |
| 通信接口 | AXI4-Lite (GP0), UART (调试输出) |

## 资源利用率

| 资源 | 使用量 | 可用量 | 利用率 |
|------|--------|--------|--------|
| LUT | ~2,800 | 17,600 | 16% |
| BRAM | 29 | 60 | 48% |
| DSP48E1 | 42 | 80 | 53% |
| 时序 | WNS > 0 | - | 满足 |

资源占用率低，剩余空间充足，可扩展摄像头采集和 HDMI 显示等外设。

## 系统架构

ARM Cortex-A9 通过 AXI4-Lite 总线直接读写 LeNet-5 IP 的寄存器。图像数据写入 IP 内部寄存器（偏移 0x400），启动推理后轮询完成标志，最后从结果寄存器（偏移 0x10）读回 10 个分类分数。

```
+------------------+         AXI4-Lite (GP0)        +-------------------+
| ARM Cortex-A9    |<------------------------------>| LeNet-5 HLS IP    |
| (PS)             |                                | (PL)              |
|                  |  0x400: 写图像 (1024 bytes)      |                   |
| main.c           |  0x000: 写启动命令               | C1: Conv+ReLU     |
| - 图像数据写入    |  0x000: 轮询完成                 | S2: MaxPool       |
| - 启动/轮询      |  0x010: 读结果 (10 int8)         | C3: Conv+ReLU     |
| - 结果输出       |                                | S4: MaxPool       |
+------------------+                                | C5: Conv+ReLU     |
                                                     | F6: FC+ReLU       |
                                                     | OUT: FC           |
                                                     +-------------------+
```

## 技术要点

### HLS 设计

- C++ 高层综合，使用 Vitis HLS 2025.2 生成 RTL
- 层序处理架构，单 MAC 分时复用，避免全并行展开导致的资源爆炸
- 8-bit 定点量化 (Q5.3 激活值, Q0.7 权重)，权重存储于片上 BRAM
- 每层独立 requantization 乘数 (M)，按 (accum * M) >> 20 进行重新量化

### 寄存器映射

通过 HLS 自动生成的硬件头文件确认实际偏移：

| 偏移 | 大小 | 用途 |
|------|------|------|
| 0x000 | 4B | CTRL: bit0=start, bit1=done, bit2=idle |
| 0x010 | 12B | out_scores[0..9]: 3 个 32-bit 寄存器，每字节一个 int8 分数 |
| 0x400 | 1024B | in_image[0..1023]: 256 个 32-bit 寄存器，每寄存器打包 4 字节 |

### 验证流程

1. **C 仿真 (csim)**: 3 张测试图片全部预测正确，确认算法正确
2. **C/RTL 协同仿真 (cosim)**: 逐周期对比 RTL 与 C 模型，位精确匹配，3/3 PASS
3. **Vivado 综合+实现**: 时序满足，资源充足
4. **硬件板级测试**: 分批测试 100 张 MNIST 图像，串口输出准确率

## 开发过程总结

项目经历多次架构迭代：

| 阶段 | 方案 | 结果 |
|------|------|------|
| 1 | 纯 Verilog RTL | FSM 调试困难，放弃 |
| 2 | HLS 全并行 | 资源超标 (LUT 1230%) |
| 3 | HLS 资源约束 | 综合通过，但 m_axi DDR 通路故障 |
| 4 | HLS s_axilite (0x10 偏移) | IP 可访问，结果全零 |
| 5 | **HLS s_axilite (0x400/0x10)** | **正确运行** |

关键踩坑: HLS 寄存器偏移由工具自动分配，需查阅生成的 `xlenet5_accel_hw.h` 确认实际映射；Zynq PS 与 PL 的 DDR 地址空间存在基地址偏移 (CPU: 0x00100000, HP: 0x00000000)；ARM 数据缓存需显式刷新才能被 PL 端读取。

最终的纯 s_axilite 方案避免了所有 DDR 缓存一致性问题和地址映射问题，数据通路简洁可靠。
